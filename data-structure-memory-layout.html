One rendered cache entry The session stores only one CachedRenderedDocument, not a per-tab map. A path mismatch forces a fresh render.
Snapshots avoid holding the lock workspace_session::snapshot clones the active document, tabs, and recent paths, then payload assembly works from that copy.
DTOs are camelCase serde(rename_all = "camelCase") makes the Rust payload line up with the frontend WorkspacePayload interface.

Sources: src-tauri/src/state.rs (AppState, AppSession, cache helpers), src-tauri/src/workspace_session.rs (WorkspaceSnapshot, tab DTO construction), src-tauri/src/workspace_payload.rs (WorkspacePayload, explorer tree DTOs), and src-tauri/src/markdown.rs (RenderedDocument).
